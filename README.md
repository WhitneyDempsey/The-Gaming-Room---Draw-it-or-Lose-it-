# The-Gaming-Room---Draw-it-or-Lose-it-
CS 230 Operating Platforms 

This respository contains my completed software design document for The Gaming Room client. 

## Project Overview 

The Gaming Room wanted to expand their existing Android game, *Draw It or Lose It*, into a web-based application that could operate across multiple platforms and devices. The software needed to support multiple platforms and devices. The software needed to support multiple teams and players, maintain consistent game data, scale to a growing user base, and provide reliable performance in a distributed environment. 

## What I did well

One area I performed particularly well in was evaluating multiple operating platforms and recommending the best overall solution for the client. I compared Linux, Windows, macOS, and mobile enviornments based on hosting capability, cost, scalability, performance, and ease of development. 

I also clearly documented technical concepts such as memory management, storage planning, and security in a way that could be understood by both technical and nontechnical audiences. 

## What was helpful in the design process

Working through the design document befor coding helps organize requirements early and think through the full system architecture. Planning items such as server structure, object relationships, memory usage, and data storage made development decisions easier later and helped identify future risks such as scaling challanges, network latency, and securty concerns before implementation. 

## What would I revise 

If i were to revise this project, I would improve the visual modeling portion by adding more system diagrams such as deployment diagrams, database relationship diagrams, or network architecture views. These additions would make the documentation even stronger and easier for stakeholders to understand. 

## Understanding User needs 

I interpreted the clients needs by focusing on performance, scalability, ease of access, and consistent multiplayer gameplay across devices. This led to recommending a Linux server enviornment with a browser based client model so users could access the game from many platforms. Considering user needs is essential because software must solve business problems while also being reliable, efficient, and easy to use. 

## My design approach 

My approach to software design included gathering requirements, comparing technical solutions, evaluating system constraints, and selecting the most practical architecture. I also used object oriented principles such as inheritance, encapsulation, and modular design to create a maintainable system structure. In future projects, I would continue using structured planning. UML modeling, scalability analysis, and security first thinking when designing software.
